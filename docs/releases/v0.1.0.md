# Skill Cortex v0.1.0 Release Notes

Skill Cortex v0.1.0 is the first public product release for this repository.

## Summary

Skill Cortex packages specialized LoRA coding skills as self-describing
artifacts, composes those artifacts into deterministic runtime bundles, and
runs local inference, serving, and bounded agent workflows against the same
runtime core.

This release does not add new runtime capabilities. It hardens the repository
for public open-source use and makes the product surface legible to first-time
visitors.

## Highlights

- Canonical public identity: Skill Cortex
- Product-first README with one clear install and demo story
- Architecture docs for Skill Factory, Skill Composer, Runtime Core, and Agent
  Runtime
- No-model end-to-end demo using checked-in fixtures and dry-run execution
- OSS release hygiene: license, changelog, contribution guide, issue templates,
  pull request template, and CI workflows

## Included product workflow

```text
Skill Factory
  -> Skill Package
  -> Skill Composer
  -> Runtime Bundle
  -> Runtime Core
  -> Agent Runtime
```

## What to try first

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -e '.[test]'
python -m skillcortex --help
python scripts/run_skillcortex_demo.py
```

## Limitations

- The documented demo flow does not retrain models or download model weights
- Demo inference and agent execution rely on dry-run validation of routing and
  control flow
- `compose-skills` supports only the routed strategy in v0.1
- The compatibility server is intentionally minimal and non-streaming
- Agent Runtime is bounded and local, not a general IDE agent

## Compatibility and research surface

The legacy `skill-lattice` research CLI remains available for backward
compatibility, but `skillcortex` is the canonical public entry point.
